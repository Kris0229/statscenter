## 1. 系統概述

### 1.1 目的
提供業餘棒壘球聯盟一套完整的成績管理平台,涵蓋球隊建置、球員名單管理、逐場成績登錄、投打排行榜查詢、Boxscore 比賽記錄、媒體(照片/影片)上傳與戰報產出。

### 1.2 範圍
- 帳號與權限管理(三級角色)
- 球隊與球員名單管理(手動 / Excel 批次匯入)
- 名單異動申請與審核流程
- 成績輸入(逐場、逐球員)
- 記錄查詢(聯盟排行、球隊排行、個人記錄、比賽 Boxscore)
- 媒體上傳與戰報產出

### 1.3 名詞定義
| 名詞 | 說明 |
|---|---|
| Admin | 聯盟管理人員,系統最高權限 |
| Power User | 球隊隊長,單一球隊之管理權限 |
| User | 一般球員 |
| Boxscore | 單場比賽完整記錄表(逐局比分、雙方打擊/投手成績) |
| 規定打席 / 規定局數 | 進入率化排行(AVG、ERA 等)所需之最低門檻,由聯盟參數設定 |

## 2. 角色與權限

> 本系統為多聯盟平台:一套系統可同時服務多個聯盟。各聯盟資料完全隔離(球隊、球員、成績、排行榜互不相通)。角色分為四級,最上層為跨聯盟的系統管理員。

### 2.1 角色定義

**System Admin(系統管理員,super_admin)** — 跨聯盟層級
- 開立/停用聯盟帳號
- 為每個聯盟建立第一個 Admin 帳號
- 管理系統層級設定
- 不參與各聯盟的日常運作(不輸入成績、不管理名單)

**Admin(聯盟管理人員)** — 單一聯盟層級
- 開創/停用球隊
- 匯入球隊名單(手動或 Excel 批次)
- 輸入/修改比賽成績
- 審核 Power User 提出之名單異動申請
- 上傳比賽照片、影片
- 產出戰報
- 管理聯盟參數(賽季、規定打席/局數、每場局數等)
- 帳號管理(於本聯盟內建立/停用帳號、指派角色)

**Power User(球隊隊長)** — 單一球隊層級
- 申請修改球員名單(改名、改背號、增加、刪除)— 需 Admin 同意後生效
- 觀看全隊與球員成績
- 上傳比賽照片、影片
- 管理本隊球員之基本資料建議(透過申請流程)

**User(球員)** — 個人層級
- 觀看球隊成績、聯盟排行、個人記錄
- 上傳比賽照片、影片
- 更新自己的個人照片、大頭照

> 除 System Admin 外,每位使用者僅隸屬單一聯盟,其所有操作自動限定在該聯盟範圍內。跨聯盟存取視為安全缺陷,一律拒絕(回傳查無此資源,不洩漏存在與否)。

### 2.3 帳號管理
- 登入方式:帳號密碼(Email);可選支援 LINE / Google OAuth(第二階段)
- 帳號建立:
  - System Admin 由系統初始化(一次性 bootstrap 腳本)建立,不經 API
  - 聯盟由 System Admin 手動開立(不提供自助申請流程)
  - Admin 由 System Admin 於開立聯盟時建立第一位,後續可由該聯盟 Admin 增設
  - Power User 由 Admin 於球隊建立時指定,或後續變更
  - User 帳號可由名單匯入時批次產生(以 Email 或手機為帳號,寄送啟用連結),或由球員自行註冊後綁定球員資料(需 Admin 或 Power User 核可綁定)
- 密碼原則:至少 8 碼,支援忘記密碼重設
- 帳號狀態:啟用 / 停用;球員轉隊或退出時保留歷史成績,帳號可停用

## 3. 球隊與球員名單管理

### 3.1 開創球隊(Admin)
欄位:球隊名稱(必填、聯盟內唯一)、隊徽圖檔、隊長(Power User 帳號)、所屬分組/聯盟、成立日期、狀態(active/inactive)。

### 3.2 球員名單建檔

**手動建檔**:單筆輸入球員資料。

**Excel 批次匯入**:
- 提供標準範本下載(.xlsx)
- 範本欄位:

| 欄位 | 必填 | 格式/驗證 |
|---|:---:|---|
| 背號 | ✔ | 0–99 整數,同隊不可重複 |
| 姓名 | ✔ | 文字 |
| 守備位置 | 選填 | P/C/1B/2B/3B/SS/LF/CF/RF/DH/UT(可複選,逗號分隔) |
| 投打習慣 | 選填 | 右投右打/左投左打…(R/L/S) |
| Email | 選填 | Email 格式(供帳號建立) |
| 手機 | 選填 | 數字 |
| 出生年月日 | 選填 | YYYY/MM/DD |

- 匯入流程:上傳 → 系統驗證 → 顯示預覽(含錯誤列標示與原因)→ 確認匯入
- 錯誤處理:格式錯誤、背號重複、必填缺漏,逐列列出錯誤原因,可下載錯誤報告修正後重傳
- 匯入模式:新增(append)/ 覆蓋整隊名單(replace,需二次確認)

### 3.3 名單異動申請流程(Power User)
1. Power User 提出申請,類型:修改姓名、修改背號、新增球員、刪除球員
2. 申請狀態:待審核 → 已核准 / 已退回(退回需填寫原因)
3. Admin 於審核後台逐筆或批次核准/退回
4. 核准後系統自動套用異動並寫入異動歷程(audit log)
5. 通知:申請提出、核准、退回時以站內通知(可選 Email)告知雙方
6. 刪除球員時,其歷史成績保留並仍計入球隊/聯盟歷史記錄,球員標記為「已離隊」

## 4. 成績記錄與查詢功能

> 說明:記錄項目與輸入欄位以台北市快慢速壘球協會(RTBA)標準計分紙為對齊基準,確保記錄員可將紙本統計格逐列對應補登。

**打擊記錄**
- 顯示(排行榜/個人頁):G、PA、AB、R、H、2B、3B、HR、RBI、BB、SO、SB、CS、AVG、OBP、SLG、OPS
- 原始輸入(對齊 RTBA 紙本):PA、AB、SH、SF、BB、HP、IO、TIE、R、H、2B、3B、HR、RBI、SO、SB、CS、GIDP、E
  - PA 分解遵循紙本驗算式:PA = AB + SH + SF + BB + HP + IO + TIE
  - 欄位對照:HP = 觸身球(HBP)、IO = 妨礙上壘/故意四壞等非打數上壘、TIE = 其他特殊上壘(依協會定義)

**投手記錄**
- 顯示(排行榜/個人頁):W、L、ERA、G、GS、CG、SHO、SV、SVO、IP、H、R、ER、HR、HB、BB、SO、WHIP、被打擊率(AVG)
- 原始輸入(對齊 RTBA 紙本):決勝(W/L/SV/HLD)、IP、NP(投球數)、BF(打席)、AB(打數)、H、HR、BB、HP、SO、R、ER、WP(暴投)、犯規(選填)

### 4.2 衍生數據計算公式
| 記錄 | 公式 |
|---|---|
| AVG(打擊率) | H ÷ AB |
| OBP(上壘率) | (H + BB + HBP) ÷ (AB + BB + HBP + SF) |
| SLG(長打率) | 壘打數 TB ÷ AB;TB = 1B + 2×2B + 3×3B + 4×HR |
| OPS | OBP + SLG |
| ERA(防禦率) | ER × 每場正規局數 ÷ IP(每場正規局數為聯盟參數,棒球 9、壘球 7) |
| WHIP | (BB + H) ÷ IP |
| 被打擊率 | 被安打 H ÷ 對方打數(對方 AB = BF − BB − HBP − SF − SH − 妨礙等) |
| IP 表示 | 以「出局數」儲存(1 局 = 3 出局),顯示為 X.0 / X.1 / X.2 |

> 註:為使 OBP、被打擊率可正確計算,輸入介面必含 PA、HBP、SF、SH、BF 等原始欄位(見第 5 章)。

### 4.3 聯盟投打排行榜
- 分「打擊 / 投手」兩個頁籤(參考 MLB 排行榜介面)
- 篩選條件:年度(賽季)、賽事類型(例行賽/季後賽)、球隊、期間(全季/區間)、守備位置
- **點擊任一欄位名稱即依該欄位排序**,再次點擊切換升冪/降冪,排序欄位以視覺高亮標示
- 率化數據(AVG、OBP、SLG、OPS、ERA、WHIP)排序時,預設隱藏未達規定打席/局數之球員,並顯示提示文字(可切換顯示全部)
  - 規定打席 = 球隊已賽場次 × 係數(預設 2.0,聯盟參數可調)
  - 規定局數 = 球隊已賽場次 × 係數(預設 1.0,聯盟參數可調)
- 顯示名次、球員姓名(連結至個人頁)、守備位置、球隊、各項數據;同分並列名次
- 支援分頁與匯出(CSV/Excel)

### 4.4 球隊投打排行榜
- 選擇單一球隊,顯示該隊所有球員之打擊/投手數據排行
- 排序、篩選行為與聯盟排行榜一致
- 附球隊累計數據(Team Totals)列

### 4.5 球員個人記錄
- 個人頁:大頭照、背號、球隊、守備位置、投打習慣
- 生涯 / 各賽季累計數據(打擊、投手分列)
- 逐場成績明細(Game Log):日期、對戰、當場數據,點擊可進入該場 Boxscore
- 個人相簿(其上傳或被標記之照片/影片)

### 4.6 比賽記錄(Boxscore)
比照 MLB Gameday Boxscore 版式,單場頁面包含:

1. **比賽資訊**:日期、時間、場地、賽事類型、主審/記錄員(選填)
2. **Line Score(逐局比分)**:雙方每局得分 + R / H / E 總計
3. **打擊成績表(雙方各一)**:棒次、球員(含替補縮排標示)、守位、AB、R、H、RBI、BB、SO、AVG(至該場為止之球季累計)
4. **打擊註記**:2B、3B、HR(含局數與出局數)、SB、CS、SF、SH、HBP、GIDP、殘壘(LOB)
5. **投手成績表(雙方各一)**:球員、IP、H、R、ER、BB、SO、HR、HBP、ERA(球季累計);標示 W / L / SV / BS / H(中繼)
6. **比賽媒體**:該場照片、影片牆
7. **戰報連結**(若已產出)
- 提供列印友善版與分享連結

## 5. 成績輸入介面(Admin)

### 5.1 設計原則
- 以「逐場 × 逐球員」為單場彙總輸入單位(非逐打席 play-by-play),輸入原始事件計數,所有率化數據由系統計算,不允許手動輸入 AVG/ERA 等衍生值,確保資料一致性。
- **作業情境:賽後電腦補登。** 記錄員賽後依 RTBA 紙本計分紙逐列補登,故輸入欄位順序與分類完全對齊紙本統計格,使補登動作等同「橫向照抄」以降低轉換摩擦。
- **裝置:桌機/筆電優先。** 全表格支援鍵盤 Tab / 方向鍵連續移動,適合鍵盤快速輸入。

### 5.2 建立比賽
欄位:日期、時間、場地、主隊/客隊、賽事類型、每場正規局數(預設帶入聯盟參數)、比賽狀態(排定/進行中/完賽/延賽/取消)。

### 5.3 打擊成績輸入
- 介面:表格式(類 Excel Grid),列 = 球員(自球隊名單挑選並排定棒次,支援替補),欄 = 數據項
- 輸入欄位(原始值,欄位順序對齊 RTBA 紙本):

| 分區 | 欄位 | 說明 |
|---|---|---|
| 打席分解 | PA | 打席(系統驗算,亦可直接帶入紙本值比對) |
| | AB | 打數 |
| | SH | 犧牲觸擊 |
| | SF | 高飛犧牲打 |
| | BB | 四壞球(含 IBB;故意四壞可另設選填欄) |
| | HP | 觸身球(即 HBP) |
| | IO | 妨礙上壘 / 非打數上壘(依協會定義) |
| | TIE | 其他特殊上壘(依協會定義) |
| 成績 | R / H | 得分 / 安打 |
| | 2B / 3B / HR | 二 / 三 / 全壘打 |
| | RBI | 打點 |
| | SO | 三振 |
| | SB / CS | 盜壘成功 / 失敗 |
| | GIDP | 雙殺打(選填) |
| | E | 失誤(守備,計入 Line Score E,選填) |

- 即時驗證規則(對齊紙本驗算欄):
  - **打席分解:PA = AB + SH + SF + BB + HP + IO + TIE**(不符時該列標紅並列出球員)
  - H ≥ 2B + 3B + HR
  - 全隊 R 合計 = Line Score 該隊總得分
  - 全隊 H 合計 = Line Score 該隊 H
  - **得分檢核:R + LOB(殘壘)+ PO(出局)= 總打席消化數(SUM)**(對齊紙本 R+LOB+PO 印章欄)
- 支援鍵盤快速輸入(Tab/方向鍵移動)、單列複製、暫存草稿

### 5.4 投手成績輸入
- 依出賽順序輸入每位投手:

| 欄位 | 說明 |
|---|---|
| 決勝標記 | W / L / SV / BS / HLD / SVO(單場 W、L 各僅一人) |
| IP | 投球局數(以 0.1 = 1 出局輸入,例 5.2) |
| NP | 投球數(對齊紙本投手欄) |
| BF | 面對打者數(打席) |
| AB | 對方打數 |
| H / HR | 被安打 / 被全壘打 |
| BB / HP | 保送 / 觸身球 |
| SO | 三振 |
| R / ER | 失分 / 自責分 |
| WP | 暴投(選填) |
| GS / CG / SHO | 先發 / 完投 / 完封(系統可依 IP 與失分輔助判定) |

- 驗證規則:
  - 雙方投手 IP 合計 ≈ 比賽局數(出局數)
  - 該隊投手 R 合計 = 對方總得分
  - ER ≤ R
  - 完封(SHO)時 R 必為 0 且 CG = 1

### 5.5 Line Score 輸入
逐局輸入雙方得分,系統自動加總 R,並與打擊/投手明細交叉驗證,不一致時標紅提示且不可送出定稿。

### 5.6 送出與修訂
- 狀態:草稿 → 定稿(定稿後計入排行榜與累計)
- 定稿後修改需 Admin 權限,系統記錄修改歷程(欄位、修改前後值、時間、操作者)
- 定稿即自動更新:球員球季累計、球隊累計、聯盟排行榜(即時或排程重算擇一,建議即時)

## 6. 媒體上傳與戰報

### 6.1 照片 / 影片上傳
- 所有角色皆可上傳;上傳時須關聯至「某場比賽」(可另選標記球員)
- 照片:JPG/PNG/HEIC,單檔 ≤ 20 MB,自動產生縮圖
- 影片:MP4/MOV,單檔 ≤ 500 MB(或改採 YouTube 連結嵌入,聯盟參數擇一)
- 內容管理:Admin 可下架不當內容;上傳者可刪除自己的檔案
- 大頭照:User 於個人頁更新,建議裁切為 1:1,顯示於名單、排行榜、Boxscore

### 6.2 戰報產出(Admin)
- 以單場比賽為單位,系統自動帶入:比分、Line Score、勝敗投手、單場亮點(自動偵測:HR、多安、雙位數三振等)
- Admin 可編輯文字內容、挑選封面照片
- 輸出:網頁版戰報頁 + 分享連結;可選匯出 PDF / 圖卡(第二階段)

## 7. 資料模型(概要)

> 多聯盟隔離:除 leagues 與 audit_logs 外,所有業務資料表皆帶 league_id 外鍵;每筆查詢一律以使用者所屬 league_id 過濾。media / reports 透過 game_id 間接歸屬聯盟,不另掛 league_id。詳細 DDL 見 BUILD_SPEC.md 第 3 章。

| 資料表 | 主要欄位 |
|---|---|
| leagues | id, name, slug, logo, status |
| users | id, league_id(super_admin 為空), email, password_hash, role(super_admin/admin/power/user), status |
| teams | id, league_id, name, logo, captain_user_id, status |
| players | id, league_id, team_id, user_id(可空), person_id(未來跨聯盟預留), name, number, positions, bats/throws, photo, status(active/left) |

## 8. 非功能需求

- **平台**:RWD 網頁,手機優先(現場輸入成績多為手機/平板)
- **效能**:排行榜查詢 < 2 秒(彙總表 + 索引);成績定稿重算 < 5 秒
- **安全**:HTTPS、角色權限伺服端驗證、上傳檔案類型白名單與掃描、個資(Email/電話)僅 Admin 與本人可見
- **稽核**:名單異動、成績修訂全程留存 audit log
- **備份**:資料庫每日備份,媒體檔案物件儲存(如 S3 相容服務)
- **多賽季**:資料以賽季分層,跨季生涯數據可累計查詢
- **匯出**:排行榜、個人記錄、Boxscore 支援 CSV/Excel 匯出
